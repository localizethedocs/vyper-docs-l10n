概覽¶

Vyper 是一種 Python 式的智慧合約語言，它可以編譯成以太坊虛擬機（EVM）位元組碼。它優先考慮安全性、可稽核性和簡潔性 。

原則¶

  • 安全性：建置安全的智慧合約應該是自然而然的事情，而不是一場艱苦的戰鬥。

  • 簡潔性：語言和編譯器都應該易於理解。

  • 可稽核性：程式碼應盡可能易於閱讀。對讀者而言，簡潔性比對編寫者而言的便利性更為重要。

主要功能¶

預設安全性

  • 陣列存取和算術運算的邊界與溢位檢查

  • 透過 @nonreentrant 裝飾器提供重入保護（請參閱 控制結構）

  • 強型別與明確的 型別轉換

可預測的執行

  • 可決定的 gas 消耗：每個函數呼叫都有可計算的上限

  • 僅允許有界迴圈（編譯時期最大迭代次數）

  • 無遞迴：執行流程在結構上是遞減的

清晰的程式碼重用

  • 模組匯入 而非類別繼承

  • 明確的 extcall 和 staticcall 關鍵字用於外部合約互動

  • 支援無法修改狀態的 純函數

編譯器強制執行的安全機制¶

Vyper 透過排除導致危險模式的功能來消除整個漏洞類別：

排除的功能

為何重要

內聯組合語言

保留型別安全、溢位保護和變數使用的可搜尋性

類別繼承

消除哪些程式碼會被執行的歧義，並簡化稽核

修改器

所有檢查都是內聯且可見的，沒有隱藏的前置/後置條件

函式覆載

函數呼叫是明確的；foo(x) 始終表示相同的意思

運算子覆載

算術運算子完全按照其表面意義執行

無限迴圈

燃料成本始終是有上限且可預測的

遞迴呼叫

呼叫圖簡單，且 gas 限制可強制執行

這些限制意味著開發者無法意外引入危險模式，即使在時間壓力或區塊鏈經驗有限的情況下也是如此。

深度驗證¶

Vyper 的設計使深度驗證在生產級智慧合約語言上變得可行。

請參閱 深度驗證 以了解驗證深度、驗證差距以及 Vyper 的形式化語意和編譯器驗證工作的目前狀態的完整討論。

十進位定點數¶

Vyper 使用十進位（非二進位）定點數。這確保像 0.1 這樣的字面值具有精確的表示，避免了二進位浮點算術中常見的細微波動精度誤差。
