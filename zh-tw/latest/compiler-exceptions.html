編譯器例外¶

當編譯合約時遇到問題，Vyper 會引發以下一或多個例外。

在可能的情況下，例外會包含一個來源碼標明，顯示程式碼中錯誤的位置：

vyper.exceptions.VariableDeclarationException: line 79:17 Persistent variable undeclared: highstBid
     78     # If bid is less than highest bid, bid fails
---> 79     if (value <= self.highstBid):
-------------------------^
     80         return False
exception ArgumentException¶

當使用無效引數呼叫函數時引發，例如位置引數數量不正確或無效的關鍵字引數。

exception CallViolation¶

當進行不合法的函數呼叫時引發，例如嘗試在兩個外部函數之間互相呼叫。

exception ArrayIndexException¶

當陣列索引超出範圍時引發。

exception EventDeclarationException¶

當事件宣告無效時引發。

exception EvmVersionException¶

當合約包含無法在目前啟用的 EVM 規則集下執行的動作時引發。

exception FunctionDeclarationException¶

當函數宣告無效時引發，例如因為不正確或不匹配的傳回值。

exception ImmutableViolation¶

當嘗試變更無法更改的變數、常數或定義時引發。例如，嘗試更新常數，或嘗試指派值給函數定義。

exception InterfaceViolation¶

當介面未被完整實作時引發。

exception InvalidAttribute¶

當參考到不存在的屬性時引發。

exception InvalidLiteral¶

當無法為字面值找到有效的型別時引發。

#pragma enable-decimals

@external
def foo():
    bar: decimal = 3.123456789123456789

此範例引發 InvalidLiteral，因為給定的字面值有太多小數位數，因此無法指派任何有效的 Vyper 型別。

exception InvalidOperation¶

當對給定型別使用無效的運算子時引發。

@external
def foo():
    a: String[10] = "hello" * 2

此範例引發 InvalidOperation，因為字串型別無法進行乘法運算。

exception InvalidReference¶

當對現有定義進行無效的參考時引發。

baz: int128

@external
def foo():
    bar: int128 = baz

此範例引發 InvalidReference，因為 baz 是一個儲存變數。對它的參考應寫成 self.baz。

exception InvalidType¶

當對給定型別使用無效的字面值時引發。

@external
def foo():
    bar: int128 = 3.5

此範例引發 InvalidType，因為 3.5 是有效的字面值，但無法轉型為 int128。

exception IteratorException¶

當迭代器建構或使用不正確時引發。

exception JSONError¶

當編譯器的 JSON 輸入格式錯誤時引發。

exception NamespaceCollision¶

當嘗試將變數指派給已被使用的名稱時引發。

exception NatSpecSyntaxException¶

當合約包含無效的 NatSpec 文件字串時引發。

vyper.exceptions.SyntaxException: line 14:5 No description given for tag '@param'
     13     @dev the feet are sticky like rice
---> 14     @param
-------------^
     15     @return always True
exception NonPayableViolation¶

當嘗試從未被標記為 @payable 的函數中存取 msg.value 時引發。

@external
def _foo():
    bar: uint256 = msg.value
exception OverflowException¶

當數值超出給定型別的範圍時引發。

exception StateAccessViolation¶

當嘗試在僅限檢視或無狀態的上下文中執行修改動作時引發。例如，在 @view 函數中寫入儲存，或在 @pure 函數中讀取儲存。

exception StructureException¶

當語法可解析，但在某些方面無效時引發。

vyper.exceptions.StructureException: line 181:0 Invalid top-level statement
     180
---> 181 '''
---------^
     182
exception SyntaxException¶

當語法無效且無法解析時引發。

vyper.exceptions.SyntaxException: line 4:20 invalid syntax
        3 struct Bid:
---> 4   blindedBid bytes32
---------------------------^
        5   deposit: uint256
exception TypeMismatch¶

當嘗試在兩個或多個型別不相容的物件之間執行動作時引發。

#pragma enable-decimals

@external
def foo():
    bar: int128 = 3
    baz: decimal = 4.2

    if baz + bar > 4:
        pass

bar 的型別是 int128，而 baz 的型別是 decimal，因此嘗試將它們相加會引發 TypeMismatch。

exception UndeclaredDefinition¶

當嘗試存取尚未宣告的物件時引發。

exception VariableDeclarationException¶

當變數宣告無效時引發。

vyper.exceptions.VariableDeclarationException: line 79:17 Persistent variable undeclared: highstBid
     78     # If bid is less than highest bid, bid fails
---> 79     if (value <= self.highstBid):
-------------------------^
     80         return False
exception VersionException¶

當合約版本字串格式錯誤或與目前編譯器版本不相容時引發。

exception ZeroDivisionException¶

當發生除以零或取餘數為零的情況時引發。
