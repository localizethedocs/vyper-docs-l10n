安裝 Vyper¶

深吸一口氣，按照說明操作，如果遇到任何錯誤，請`建立一個 Issue <https://github.com/vyperlang/vyper/issues>`_。

小訣竅

剛接觸 Vyper？請從 快速開始 指南開始，快速讓專案執行起來。

備註

試驗此語言最簡單的方式是使用線上編譯器：

  • Try Vyper!：由 Vyper 團隊維護，需要 GitHub 登入

  • Remix：由以太坊基金會維護，在 Plugin 管理器中啟用 vyper-remix 外掛

獨立安裝¶

Vyper CLI 可以使用 uv tool 或 pipx 安裝。如果您尚未安裝這些工具，請先造訪它們的安裝頁面：

  • https://github.com/astral-sh/uv

  • https://github.com/pypa/pipx

然後安裝 Vyper：

uv tool install vyper

或者：

pipx install vyper

二進位檔¶

或者，可以在 GitHub releases 頁面下載適用於 Windows、Mac 和 Linux 的預編譯 Vyper 二進位檔：https://github.com/vyperlang/vyper/releases。

PIP¶

安裝 Python¶

Vyper 只能使用 Python 3.11 及更高版本建置。如果您需要了解如何安裝正確版本的 Python，請遵循官方 Python 網站 的指示。

建立虛擬環境¶

由於 pip 安裝預設不是隔離的，此安裝方法適用於更有經驗的 Python 開發者，他們將 Vyper 作為函式庫使用，或希望在其他具有 pip 相依性的 Python 專案中使用它。

**強烈建議**將 Vyper 安裝在**虛擬 Python 環境**中，這樣新安裝的套件和建置的相依性將嚴格包含在您的 Vyper 專案中，不會改變或影響您其他的開發環境設定。

備註

要了解更多關於虛擬環境的資訊，請參閱：virtualenv 指南。

安裝 Vyper¶

每個標記版本的 vyper 都會上傳到 pypi，並可以在虛擬環境中安裝：

pip install vyper

或者：

uv pip install vyper

要安裝特定版本，請使用：

pip install vyper==0.4.0

您可以透過在終端機/命令提示字元中輸入以下內容來檢查 Vyper 是否已完全安裝：

vyper --version

Docker¶

Vyper 可以從 dockerhub 下載為 docker 映像檔：

docker pull vyperlang/vyper

要執行編譯器，請使用 docker run 命令：

docker run -v $(pwd):/code vyperlang/vyper /code/<contract_file.vy>

或者，您可以登入 docker 映像檔並在提示符中執行 vyper。

docker run -v $(pwd):/code/ -it --entrypoint /bin/bash vyperlang/vyper
root@d35252d1fb1b:/code# vyper <contract_file.vy>

也支援一般參數，例如：

docker run -v $(pwd):/code vyperlang/vyper -f abi /code/<contract_file.vy>
[{"stateMutability": "nonpayable", "type": "function", "name": "test1", "inputs": [{"name": "a", "type": "uint256"}, {"name": "b", "type": "bytes"}], "outputs": []}, {"stateMutability": "nonpayable", "type": "function", "name": "test2", "inputs": [{"name": "a", "type": "uint256"}], "outputs": []}]

備註

如果您想了解如何安裝 Docker，請遵循其 文件。

nix¶

在 nix package search 查看透過 nix 支援的版本

備註

Vyper 的 derivation 位於 nixpkgs

安裝 Vyper¶

nix-env -iA nixpkgs.vyper
