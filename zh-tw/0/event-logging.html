事件日誌紀錄¶

Vyper 可以記錄事件，以便由使用者介面捕獲和顯示。

日誌紀錄範例¶

此範例取自 ERC20 合約範例，展示了事件日誌紀錄的基本流程：

# Events of the token.
event Transfer:
    sender: indexed(address)
    receiver: indexed(address)
    value: uint256

event Approval:
    owner: indexed(address)
    spender: indexed(address)
    value: uint256

# Transfer some tokens from message sender to another address
def transfer(_to : address, _value : uint256) -> bool:

   ... Logic here to do the real work ...

   # All done, log the event for listeners
   log Transfer(msg.sender, _to, _value)

讓我們看看這段程式碼在做什麼。

  1. 我們宣告了兩個要記錄的事件型別。這兩個事件相似之處在於都包含兩個已索引的地址欄位。已索引欄位不構成事件資料本身的一部分，但可以由想要捕獲事件的客戶端進行搜尋。此外，每個事件都包含一個資料欄位，在每個案例中稱為 value。事件可以包含多個任意名稱的引數。

  2. 在 transfer 函數中，在我們執行完必要的工作後，我們記錄事件。我們傳遞三個引數，對應 Transfer 事件宣告的三個引數。

監聽事件的客戶端將使用 如 web3.js 的函式庫 來宣告和處理他們感興趣的事件：

var abi = /* abi as generated by the compiler */;
var MyToken = web3.eth.contract(abi);
var myToken = MyToken.at("0x1234...ab67" /* address */);

// watch for changes in the callback
var event = myToken.Transfer(function(error, result) {
    if (!error) {
        var args = result.returnValues;
        console.log('value transferred = ', args._amount);
    }
});

在此範例中，監聽客戶端宣告了要監聽的事件。每當合約發送此日誌事件時，回呼就會被呼叫。

宣告事件¶

讓我們更詳細地看看事件宣告。

event Transfer:
    sender: indexed(address)
    receiver: indexed(address)
    value: uint256

EVM 目前有五個用於發送事件日誌的 opcode：LOG0、LOG1、LOG2、LOG3 和 LOG4。這些 opcode 可用於建立日誌記錄，其中每個日誌記錄由**主題（topics）**和**資料（data）**組成。主題是 32 位元組的「字組」，用於描述事件中發生的情況。雖然主題是可搜尋的，但資料不是。然而事件資料不受限制，這意味著您可以包含大型或複雜的資料，如陣列或字串。不同的 opcode（LOG0 到 LOG4）允許不同數量的主題。例如，LOG1 包含一個主題，LOG2 包含兩個主題，依此類推。事件宣告看起來類似於結構宣告，包含一個或多個傳遞給事件的引數。典型的事件將包含兩種引數：

  • **已索引**引數（主題），可以被監聽器搜尋。每個已索引引數由 indexed 關鍵字標識。在此處，每個已索引引數都是一個地址。您最多可以有四個已索引引數（LOG4），但已索引引數不會直接傳遞給監聽器，儘管其中一些資訊（如發送者）可能在監聽器的 results 物件中可用。

  • Value arguments (data), which are passed through to listeners. You can have any number of value arguments and they can have arbitrary names, but each is limited by the EVM to be no more than 32 bytes.

請注意，日誌記錄的第一個主題由已發生事件名稱的簽名組成，包括其參數的型別。也可以建立沒有引數的事件。在這種情況下，請使用 pass 陳述式：

event Foo: pass

日誌紀錄事件¶

Once an event is declared, you can log (send) events. You can send events as many times as you want to. Please note that events sent do not take state storage and thus do not cost gas: this makes events a good way to save some information. However, the drawback is that events are not available to contracts, only to clients.

記錄事件是使用 log 陳述式完成的：

log Transfer(msg.sender, _to, _amount)

The order and types of arguments given must match the order of arguments used when declaring the event.

監聽事件¶

在上面的監聽器範例中，result 引數實際上傳遞了`大量資訊 <https://web3js.readthedocs.io/en/v1.2.6/web3-eth-contract.html#contract-events-return>`_。這裡我們最感興趣的是 result.returnValues。這是一個屬性與事件中宣告的屬性相符的物件。請注意，此物件不包含已索引的屬性，這些屬性只能在建立回呼的原始 myToken.Transfer 中進行搜尋。
